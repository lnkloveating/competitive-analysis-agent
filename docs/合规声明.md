# 合规声明

本声明说明「AI 驱动的电竞鼠标竞品分析 Agent 协作系统」在数据采集、隐私安全、工具与模型使用方面的合规原则与具体做法。核心立场：**只采集公开信息、不绕过反爬、不抓取隐私数据、所有外部数据保留来源与可信度、抓不到就如实标记缺失而非编造。**

---

## 1. 数据来源与采集方式

| 数据 | 来源 | 采集方式 |
|------|------|---------|
| 硬件事实底座 | 厂商官网公开规格、公开整理 | 预置于本地 JSON（`data/products/`），仅相对稳定字段 |
| 候选发现 | 公开搜索结果 | **Tavily 官方搜索 API**（合规检索服务），非自建爬虫扫站 |
| 官网规格 | 厂商官方产品页 | 抓取公开页面 → LLM 结构化抽取 |
| 实时价格 | 官方店 / 公开电商页 | 公开页面，分级可信 |
| 用户评价 / 博主测评 | 公开评测站 / 社区 / 视频公开页 | 公开正文 / 公开字幕，结构化为体验信号 |

- 所有外部数据均为**公开可访问信息**，不涉及登录态、付费墙、私域内容。
- 候选发现优先走 **Tavily 官方 API**，由其负责底层抓取的合规性；直接取正文时使用标准 User-Agent，不伪装、不模拟登录。

---

## 2. robots.txt 与反爬合规

- **遵守目标站点服务条款**：仅访问公开页面；不破解验证码、不模拟登录、不绕过付费墙。
- **反爬不绕过、只降级标记**：
  - 价格被拦截 → 标记 `official_price_blocked`，并在报告可信度中扣分；
  - 评价/正文抓取失败 → 记入 `blocked_sources`，对应维度如实标记「未抓取到数据」；
  - 绝不通过破解或伪装手段强行获取被站点拒绝的内容。
- **取正文代理**：对部分公开页面使用 reader 代理（Jina Reader）获取可读正文，仅用于读取公开内容，不用于规避访问控制。
- **后续增强**：可在直接抓取前增加 `robots.txt` 程序化预检（按 path 判定 allow/deny），进一步前置合规判断。

---

## 3. 数据隐私与安全

- **无个人隐私数据**：系统不采集、不存储任何用户访谈、问卷或个人身份信息；本项目不涉及 PII。
- **人工输入仅限当前任务**：答辩现场的人工反馈只作为**当前分析任务**的 evidence，默认低可信、待验证，不做跨任务持久化、不并入训练数据。
- **密钥不入仓**：所有 API Key 仅存于 `backend/.env`，已被 `.gitignore` 忽略，不提交到 Git 仓库。
- **来源留痕**：每条外部数据保留 `source_url` 与 `credibility` 标记，可回溯到原始来源。

---

## 4. 工具、模型与 API 使用合规

| 类别 | 使用 | 说明 |
|------|------|------|
| 大模型 | DeepSeek（OpenAI 兼容协议） | 仅用于「非结构化文本 → 结构化抽取/总结」，不作为事实来源；输出须基于给定证据 |
| 检索 | Tavily Search API | 官方合规检索服务 |
| 取正文 | Jina Reader | 仅读取公开页面正文 |
| 可观测 | LangSmith | 记录 Agent trace 与 token，便于审计 |

- 模型、检索、取正文服务均按各自官方协议、在授权配额内调用。
- LLM 被严格约束为「抽取器 / 总结器」：不能凭空补字段、结论必须绑定 `evidence_id`，由 VerificationAgent 校验、QualityAgent 降级。

---

## 5. 可信度分级与风险披露

- **分级可信**：官网（高） > 电商（中/低） > 视频/搜索摘要（低）；低可信来源可展示但必须标注。
- **主动披露已知限制**：
  - 视频平台字幕 / 官方 API 在 demo 阶段未接入，部分体验维度可能抓取不全；
  - 电商价格存在区域差异；
  - 搜索摘要不作为强证据。
- **不编造**：任何维度抓不到数据，一律标记缺失并在报告可信度中扣分，而非用假数据填充。

---

## 6. 提交材料与可审计性

- 代码、Schema、Agent 协议、架构与本声明均纳入 Git 仓库，结构清晰、可审计。
- 每次分析的 Agent 决策过程（trace_log）、证据链（evidence / claim）、质量评分（quality_result）均可在系统内查看，并可通过 LangSmith 进一步追溯。
